# Протокол документации проекта

> Шаблон и правила создания документации для разработки проектов с AI-ассистентами

---

## Философия

**Проблема:** AI не помнит предыдущие разговоры. При каждой новой сессии нужно заново объяснять контекст проекта.

**Решение:** Структурированная документация, которая читается AI в начале каждой сессии и позволяет продолжить работу без потери контекста.

**Принцип:** Один раз описал — бесконечно переиспользуешь.

---

## Структура файлов

```
project-root/
├── README.md              # Входная точка для людей
├── PROJECT-CONTEXT.md     # Входная точка для AI (ГЛАВНЫЙ ФАЙЛ)
├── IDEAS.md               # Бэклог идей и фич
├── ARCHITECTURE.md        # Архитектура и данные
├── ENTITIES-RELATIONS.md  # Справочник сущностей и связей (детальный)
├── UI-GUIDELINES.md       # Дизайн-система
├── PAGES-STATUS.md        # Статус разработки
├── WORKLOG.md             # История сессий (опционально)
└── CHANGELOG.md           # История изменений (опционально)
```

**Примечание:** ENTITIES-RELATIONS.md содержит детальное описание всех сущностей, их полей, связей и матриц доступа. ARCHITECTURE.md — упрощённая версия для быстрого понимания структуры.

## 1. README.md — Для людей

### Назначение
Входная точка для разработчиков и пользователей. Показывает в GitHub.

### Обязательные секции
```markdown
# Название проекта

Краткое описание (1-2 предложения).

## Быстрый старт

Как запустить / использовать.

## Основные возможности

3-5 ключевых фич.

## Технологии

Стек технологий.

## Структура проекта

Основные директории и файлы.

## Как внести вклад

Для open-source или командной работы.
```

### Правила
- Длина: до 100 строк
- Язык: тот же, что и у целевой аудитории
- Без технических деталей (они в других файлах)

[Матрица прав или описание ролей]

## 5. Статусы

[Справочник статусов для каждой сущности]

[Редактируемые справочники системы]

### Правила
- **Самый важный файл** — должен быть самодостаточным
- Длина: 200-400 строк
- Обновлять при каждом значимом изменении
- Содержать конкретные данные (имена, статусы, приоритеты)
- Не дублировать IDEAS.md — ссылаться на него

## 3. IDEAS.md — Бэклог идей

### Назначение
Детальное описание фич с UI-макетами, полями и логикой.

### Структура
```markdown
# Идеи для развития [Название проекта]

> Статус: 💡 Идея | 📝 В работе | ✅ Реализовано | ❌ Отклонено

## [N]. [Категория]

### [N].[M] [Название фичи] [статус]
**Приоритет:** Высокий/Средний/Низкий

**Проблема:**
[Какую проблему решаем]

**Решение:**
[Описание решения]

**Структура/Макет:**
```
[ASCII-макет интерфейса]
```

**Поля:**
| Поле | Тип | Описание |
|------|-----|----------|

**Правила:**
- [Правило 1]
- [Правило 2]

**Вопросы для уточнения:**
- [Вопрос 1]
```

### Правила
- Каждая идея — отдельный раздел с номером
- ASCII-макеты для визуализации UI
- Таблицы для полей и параметров
- Вопросы для уточнения в конце каждой идеи
- Статусы обновлять при изменении
- Приоритеты в конце файла

### Приоритеты в конце
```markdown
## Приоритеты на ближайшую реализацию

### Первый приоритет:
1. [Фича 1]
2. [Фича 2]

### Второй приоритет:
3. [Фича 3]
```

## 4.1. ENTITIES-RELATIONS.md — Справочник сущностей (детальный)

### Назначение
Единый справочник всех сущностей системы с полями, связями, статусами и матрицами доступа. Основной документ для разработчиков frontend и backend.

### Отличие от ARCHITECTURE.md
| Файл | Назначение | Детальность |
|------|------------|-------------|
| ARCHITECTURE.md | Быстрое понимание структуры | Упрощённые интерфейсы |
| ENTITIES-RELATIONS.md | Полная спецификация для разработки | Все поля, типы, связи, матрицы |

### Структура
```markdown
# Сущности и связи [Название проекта]

## 1. Обзор сущностей
[Таблица со списком всех сущностей]

## 2. ER-диаграмма
[ASCII-диаграмма связей]

## 3. Сущности (детально)

### 3.1 [Сущность]
**Назначение:** [описание]

| Поле | Тип | Обяз. | Описание |
|------|-----|-------|----------|

**Связи:**
- `relation: Entity[]` — описание связи (1:N)

**Страницы:** `XX-page.html, YY-page.html`

**UI-особенности:**
- [Особенности отображения для разных ролей]

## 4. Справочники
[Описание редактируемых справочников]

## 5. Матрица связей
[Таблица всех связей между сущностями]

## 6. Статусы сущностей
[Справочник статусов для каждой сущности]

## 7. Права доступа
[Матрицы доступа к сущностям по ролям]
```

### Правила
- Каждая сущность — отдельный раздел с номером
- Таблицы для полей и связей
- Указывать страницы, где используется сущность
- Описывать UI-особенности для разных ролей
- Матрицы доступа — обязательно

## Правила обновления

### При добавлении новой фичи
1. Добавить идею в IDEAS.md
2. Обновить приоритеты в IDEAS.md
3. Если новая сущность — добавить в ARCHITECTURE.md и PROJECT-CONTEXT.md
4. Если новый экран — добавить в PAGES-STATUS.md

### При завершении работы
1. Обновить статусы в IDEAS.md (💡 → ✅)
2. Обновить PAGES-STATUS.md
3. Добавить запись в WORKLOG.md
4. Закоммитить изменения

### При начале новой сессии
1. AI читает PROJECT-CONTEXT.md
2. AI смотрит приоритеты в PAGES-STATUS.md или IDEAS.md
3. AI читает детали реализации в IDEAS.md

## Промпт для AI

При начале работы с новым проектом используйте:

```
Я работаю над проектом [название]. 

Прочитай файл PROJECT-CONTEXT.md в корне проекта для понимания контекста.

Затем посмотри PAGES-STATUS.md — там статус разработки и приоритеты.

Детали реализации конкретных фич — в IDEAS.md.

После прочтения предложи план работы на эту сессию.
```

## Шаблон для нового проекта

При создании нового проекта попросите AI:

```
Создай структуру документации для нового проекта:

Проект: [название]
Описание: [краткое описание]
Технологии: [стек]
Тип: [прототип/веб-приложение/мобильное приложение]

Создай следующие файлы:
1. README.md — краткое описание
2. PROJECT-CONTEXT.md — полный контекст для AI
3. IDEAS.md — начни с базовых фич
4. ARCHITECTURE.md — опиши основные сущности
5. UI-GUIDELINES.md — базовая дизайн-система
6. PAGES-STATUS.md — список планируемых страниц
```

## Почему это работает

| Проблема | Решение |
|----------|---------|
| AI не помнит контекст | PROJECT-CONTEXT.md читается каждый раз |
| Идеи теряются | IDEAS.md — централизованное хранилище |
| Непонятны требования к фиче | IDEAS.md — детали, макеты, поля |
| Разные стили кода | UI-GUIDELINES.md — единый стиль |
| Непонятно, что готово | PAGES-STATUS.md — актуальный статус |
| Новые разработчики теряются | README.md + PROJECT-CONTEXT.md |
