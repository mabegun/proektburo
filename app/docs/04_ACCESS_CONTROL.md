## Проверка прав в API

### Пример зависимости FastAPI

```python
async def get_current_user(authorization: str = Header(...)) -> dict:
    """Получить текущего пользователя"""
    token = authorization.replace("Bearer ", "")
    user = verify_token(token)
    if not user:
        raise HTTPException(status_code=401, detail="Unauthorized")
    return user


def check_permission(user: dict, resource: str, action: str):
    """Проверить права доступа"""
    role = user["role"]
    
    permissions = {
        "director": {"*": ["*"]},  # Все действия ко всем ресурсам
        "gip": {
            "projects": ["read_own", "create", "update_own", "delete_own"],
            "sections": ["read_own", "create", "update_own"],
        },
        "executor": {
            "sections": ["read_assigned", "update_assigned"],
            "files": ["upload_assigned", "download", "delete_own"],
        },
        "observer": {
            "*": ["read_assigned"],
            "chat": ["read", "write"],
        }
    }
    
    user_perms = permissions.get(role, {})
    resource_perms = user_perms.get(resource, user_perms.get("*", []))
    
    if "*" in resource_perms or action in resource_perms:
        return True
    
    raise HTTPException(status_code=403, detail="Access denied")
```

## Сценарии использования

### Сценарий 1: Директор создаёт проект

1. Director → POST /api/projects → ✅ Создан
2. Director → POST /api/projects/{id}/members/{gip_email} → ✅ Добавлен ГИП
3. Director → POST /api/projects/{id}/sections → ✅ Создан раздел

---

### Сценарий 2: ГИП добавляет раздел

1. GIP → GET /api/projects/{id} → ✅ (если он ГИП этого проекта)
2. GIP → POST /api/projects/{id}/sections → ✅ Создан
3. GIP → POST /api/sections/{id}/observers/{email} → ✅ Добавлен наблюдатель

---

### Сценарий 3: Исполнитель загружает файл

1. Executor → GET /api/sections/{id} → ✅ (если он исполнитель этого раздела)
2. Executor → POST /api/upload → ✅ Файл загружен
3. Executor → POST /api/chat → ✅ Сообщение отправлено

---

### Сценарий 4: Наблюдатель пытается удалить файл

1. Observer → GET /api/files/{id} → ✅ (просмотр)
2. Observer → DELETE /api/files/{id} → ❌ 403 Forbidden

## Токены и сессии

### JWT Token

**Структура:**
```json
{
  "sub": "director@bureau.ru",
  "role": "director",
  "exp": 1710864000,
  "iat": 1710777600
}
```

**Срок действия:** 24 часа

**Хранение:** localStorage (frontend)
