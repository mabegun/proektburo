## 18. Статистика замечаний по сотрудникам 💡
**Приоритет: Высокий**

**Описание:**
Руководителю необходима статистика по каждому сотруднику для контроля качества отработки замечаний экспертизы.

**Метрики:**
1. **Общее количество замечаний** — сколько всего замечаний получал сотрудник
2. **Время отработки (по каждому замечанию)** — сколько дней прошло от создания до согласования ГИПом
3. **Среднее время отработки** — среднее арифметическое по всем замечаниям
4. **Статусы замечаний** — распределение по статусам (в работе, на проверке, закрыто)
5. **Процент замечаний с первого раза** — сколько замечаний закрыто без возврата на доработку

**Формулы:**
```
Время отработки = Дата согласования ГИПом − Дата создания замечания
Среднее время = Σ(Время отработки) / Количество закрытых замечаний
% с первого раза = (Закрыто без возвратов / Всего закрыто) × 100
```

**Где отображать:**
1. **В карточке сотрудника** (`09-employees-list.html` → детали)
   - Блок "Статистика замечаний"
   - Всего, в работе, на проверке, закрыто
   - Среднее время отработки
   - % с первого раза

2. **В списке замечаний проекта** (`04-2-project-expertise.html`)
   - Колонка "Время отработки" в таблице
   - Цветовая индикация:
     - Зелёный (< 3 дней)
     - Жёлтый (3-7 дней)
     - Красный (> 7 дней)

3. **В карточке замечания** (`16-expertise-comment.html`)
   - Время отработки для данного замечания
   - Сравнение со средним временем сотрудника
