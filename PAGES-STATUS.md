## Структура страниц раздела (обновлено)

### Структура вкладок раздела:

```
06-section-detail.html      → Обзор (информация, вводные данные, участники, договор)
06-1-section-files.html     → Файлы (список, загрузка, согласование)
06-2-section-discussion.html → Обсуждение (чат с вложениями)
06-3-section-finances.html  → Финансы (договор, выплаты)
06-4-section-expertise.html → Замечания (экспертиза)
06-5-section-history.html   → История (таймлайн изменений)
06-6-section-observer.html  → Для наблюдателя (ограниченный доступ)
```

### Структура первой страницы раздела (Обзор):

**Левая колонка (2/3):**
1. **Информация о разделе** — проект, шифр, тип, статус, даты, дедлайн
2. **Вводные данные** — текст + файлы (редактирует ГИП)

**Правая колонка (1/3):**
1. **Участники** — ответственный, соисполнители, наблюдатели
2. **Договор** — кратко (номер, сумма, статус) + ссылка на финансы
3. **Последние файлы** — 3-5 файлов + ссылка "Все"

## API Endpoints для раздела (ожидаемые)

### Разделы
- `GET /api/sections/:id` — детали раздела (обзор)
- `GET /api/sections/:id/files` — файлы раздела
- `POST /api/sections/:id/files` — загрузка файла
- `PUT /api/sections/:id/files/:fileId/approve` — согласование файла
- `GET /api/sections/:id/discussion` — обсуждение раздела
- `POST /api/sections/:id/discussion` — добавить сообщение
- `GET /api/sections/:id/finances` — финансы раздела
- `GET /api/sections/:id/expertise` — замечания раздела
- `GET /api/sections/:id/history` — история изменений
- `PUT /api/sections/:id/input-data` — обновить вводные данные
- `PUT /api/sections/:id/participants` — управление участниками
