# Проектное бюро - Прототип системы управления

Интерактивный HTML-прототип системы управления проектами для проектного бюро.

## Описание

Данный прототип представляет собой интерактивную модель системы управления проектами, разработанную для архитектурно-проектных организаций. Система позволяет управлять проектами, задачами, исполнителями и документацией в едином интерфейсе.

## Технологии

- **HTML5** - структура страниц
- **Tailwind CSS** (CDN) - стилизация интерфейса
- **Шрифт Inter** (Google Fonts)
- **Иконки Heroicons** (SVG inline)

## Функциональные возможности

### Управление проектами
- Создание и редактирование проектов
- Статусы: В работе, На согласовании, Завершён, Архив
- Привязка разделов проектной документации (АР, ГП, КР, ОВ, ВК, ЭОМ и др.)
- Назначение ответственных исполнителей
- **Индикаторы разделов на карточках проектов:**
  - Цветовая индикация статуса каждого раздела
  - Кликабельные индикаторы — переход на страницу раздела
  - Подсказки с полным названием, статусом и ответственным
  - Разделение на "Мои разделы" и "Доступны для просмотра" для исполнителей
- **Индикаторы изысканий на карточках проектов:**
  - Видны только Директору и ГИПу
  - Исполнители видят только если им открыт доступ (в observers)
  - Метка `И:` перед изысканиями для понимания типа индикатора
- **Команда проекта:**
  - Управление участниками проекта (только для руководителей)
  - Используется при назначении соисполнителей и наблюдателей
- **Быстрое добавление:**
  - Кнопка "Добавить" в блоке разделов проекта
  - Кнопка "Добавить" в блоке изысканий
  - Единообразный UI для обоих блоков

### Управление задачами
- Создание и назначение задач
- Приоритеты и сроки выполнения
- Статусы: Не начата, В работе, На проверке, Завершена
- Привязка к проектам и разделам

### Управление исполнителями
- Карточки исполнителей с должностями
- Назначение на проекты и разделы
- Контроль загрузки
- **Страница профиля** (планируется)
- **Ограниченный доступ для исполнителей:** только свой профиль в настройках

### Документооборот
- Загрузка файлов в разделы проектов
- Версионность документов
- Статусы согласования

### Справочники системы
- **Разделы проекта** — редактируемый справочник (АР, ГП, КР...)
- **Виды изысканий** — ИГД, ИГИ, ИЭИ, ИАДИ
- **Категории расходов** — для учёта затрат
- **Типы контрагентов** — ООО, ИП, Самозанятый, Физлицо, Сотрудник (с матрицей документов)

### Изыскания
- Блок изысканий на странице проекта с кнопкой быстрого добавления
- Индикаторы изысканий на карточках проектов
- Страница изыскания с блоками: статус, ответственный, исполнитель (юрлицо), файлы, observers, обсуждение
- Открытие доступа исполнителям (observers)

### Вкладки проекта
- **Обзор** — основная информация о проекте
- **Обсуждение** — комментарии участников с вложениями
- **Замечания** — все замечания экспертизы с фильтрами
- **История** — журнал изменений проекта
- **Финансы** — финансовые операции:
  - ГИП/Директор видят все расходы + сводку по авторам
  - Исполнитель видит только свои расходы

### Финансы и закрывающие документы (планируется)
- **Тип контрагента** — автоматическое определение обязательных документов
- **Договор как сквозная сущность** — виден из раздела, финансов, карточки исполнителя
- **Привязка выплат к разделам** — статистика по исполнителям
- **Загрузка документов исполнителем** — подсказки по комплектности, мотивация

### Уведомления
- Системные уведомления о событиях
- Фильтрация по типам
- Статусы прочтения

### Отчётность
- Статистика по проектам
- Загрузка исполнителей
- Экспорт данных

## Дизайн

- Минималистичный современный интерфейс
- Адаптивная вёрстка
- Цветовая схема: оттенки серого (slate)
- Шрифт: Inter

## Как использовать

1. Откройте файл `00-map.html` или `01-login.html` в браузере
2. Навигация осуществляется через боковое меню
3. Все ссылки между страницами работают
